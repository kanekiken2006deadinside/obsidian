#### Игра Akari (Light Up)

Смысл: расставить все лампочки так, чтобы все клетки были освещены, но лампочки не светили друг на друга
В игре есть четкая иерархия объектов и можно выстроить четкую логику взаимодействия между ними.
- Абстрактные классы и наследование:
Базовый абстрактный класс для всех элементов игрового поля. Он определяет общие свойства, те координаты плит, их освещенность. Наследниками будут пустые клетки, которые можно осветить, стена/черный блок, который может содержать цифру/часть картинки и лампу которую ставит юзер.
- Инкапсуляция:
Будет использ-ся чтобы скрыть внутреннюю логику проверки светит ли лампа на другую или нет (внутри класса). Можно использовать приватные атрибуты (ex: grid),  чтобы состояние поля нельзя было бы изменять через сам интерфейс игры напрямую без каких-либо проверок .
- Полиморфизм:
Метод команды для отрисовки объекта (нарисуй что-то в координатах х,у) и метод визуализации, чтобы не просто нарисовать линию, а создать целую сцену, они могут быть определены в базовом классе, но каждый наследник будет использоваться, то есть отрисовываться по-разному (ex: стена черная, лампа желтая)
- Специальные методы: 
 1) __init__: инициализация игрового поля, установка размера и сложности
 2) __str__: позволит вывести игровое поле в консоль в виде текстовых символов
 3) __getitem__/__setitem__/__delitem__: сделает класс центра игры (ктр с инкапсуляцией, знает все о правилах игры, расположении объектов, о наступлении победы) очень удобным, тк можно будет обращаться к определенной плитке/клетку как board[x,y] = Lamp(), а __delitem__ будет убирать лампу с поля
 4) сравнения  __lt__, __eq__ и тд: можно использовать для сравнения рекордов игрока (скорости времени прохождения) или для сравнения сложности разных уровней на основе количества плит/клеток и лампочек.

#### Игра судоку
Это будет очень похоже на предядущую игру в плане кода скорее всего
- **Наследование и Абстрактные классы:** 
Можно создать абстрактный класс , от ктр наследуются класс с цифрами, и изначальными данными  и класс с пустыми клетками, куда игрок вписывает недостающие цифры
- **Инкапсуляция:** 
Класс будет скрывать двумерный массив с цифрами и хранить методы для проверки правил, чтобы цифры не повторялись в блоках 3×3, строках и столбцах
- **Специальные методы:** 
1) `__getitem__` / `__setitem__`: чтобы обращаться к клетке по координатам `board[0, 5]`
2) `__eq__`: можно сравнивать две доски (например, текущую доску игрока и доску с решением)
- **Графический интерфейс:** 
Очень удобно будет делать на кнопках или текстовых полях
